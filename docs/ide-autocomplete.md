IDE Autocompletion
==================

The Problem
-----------

Components in this package are class-based. IDEs do not surface attribute-name or attribute-value completion inside `<x-...>` tags out of the box:

- The official Laravel VS Code extension completes **component names** and anonymous `@props`-based components, but not class-based component attributes.
- PHPDoc `@var`/`@param` annotations on component properties provide PHP-level IntelliSense in PHP files, but IDEs do not project this information into Blade tag attributes.
- VS Code's native `html.customData` mechanism is consumed by the HTML language service, which is **not** active for `.blade.php` files in Blade language mode — so it does not provide completion inside `<x-...>` in Blade mode.

To bridge this gap, the package ships an Artisan command that generates IDE metadata files from the registered components.

The `blade-ui-kit-bs:ide` Command
----------------------------------

The command introspects the registered components at runtime, respecting the configured `prefix` and any `merge()`, `replace()`, or `replaceAlias()` customisations, and generates metadata files.

### Basic Usage

```bash
php artisan blade-ui-kit-bs:ide
```

### Modes

**Interactive (default at a TTY)**

When run in a terminal, the command uses Laravel Prompts to ask which files to generate and where to put them. Choose one or more output files and confirm the output directory.

**Non-interactive / flag-driven**

Use `--no-interaction` to skip all prompts and generate all three files:

```bash
php artisan blade-ui-kit-bs:ide --no-interaction
```

Restrict which files are generated by combining `--no-interaction` with one or more output flags:

| Flag | File generated |
|------|---------------|
| `--snippets` | VS Code snippets only |
| `--json` | VS Code Custom Data only |
| `--ide-json` | PhpStorm IDE JSON only |

```bash
# Generate only snippets and the PhpStorm file
php artisan blade-ui-kit-bs:ide --no-interaction --snippets --ide-json
```

### Output Locations

The **VS Code** files go to `.vscode/` by default (override with `--output=DIR`). The
**PhpStorm** `ide.json` goes to `ide-helper/blade-ui-kit-bootstrap/ide.json` by default
(override with `--ide-output=DIR`).

Why a dedicated subfolder rather than the project root? Laravel Idea only reads files named
exactly `ide.json`, **scans the project recursively, and merges every `ide.json` it finds**.
Writing to a package-owned subfolder means our file never collides with an `ide.json` your
application maintains for its own routes/configs/macros — both are picked up and merged — and
it is always safe to regenerate.

```bash
# VS Code files in a custom dir, ide.json in a custom dir
php artisan blade-ui-kit-bs:ide --no-interaction --output=resources/.vscode --ide-output=.idea/blade-ui-kit
```

The Three Output Files
----------------------

### 1. VS Code Snippets — `blade-ui-kit-bootstrap.code-snippets`

**Role: fallback, zero-install.**

This file is picked up automatically by VS Code from your `.vscode/` directory without any extension required. Type a component tag prefix (e.g. `x-btn`) to trigger a snippet that scaffolds the full tag:

- Required attributes are pre-filled as tab stops.
- The `variant` attribute is offered as a value dropdown (its first value, `primary`, is the component default, so tabbing past it is harmless).

Snippets stay deliberately lean: other constrained attributes (`size`, `type`, `confirm-variant`, …) are not scaffolded, because a snippet dropdown always keeps its first value when you tab past it, which would force unwanted defaults. Full, non-forcing value completion for every attribute is the dedicated extension's job (see below). Use snippets when you cannot or do not want to install that extension.

### 2. VS Code Custom Data — `blade-ui-kit-bootstrap.html-data.json`

**Role: primary solution (requires a dedicated VS Code extension).**

This file follows the VS Code [Custom Data format](https://github.com/microsoft/vscode-custom-data) and contains attribute names, allowed values, and hover descriptions for every registered component.

Because VS Code's HTML language service is not active in Blade language mode, this file cannot be wired up via the `html.customData` setting in `settings.json` alone — a dedicated VS Code extension is needed to consume it and inject completions into `.blade.php` files. That extension is a forthcoming, separate deliverable. Once available, it will be the recommended way to get full attribute-name and attribute-value completion with hover documentation inside `<x-...>` tags.

### 3. PhpStorm / Laravel Idea — `ide-helper/blade-ui-kit-bootstrap/ide.json`

This file follows the [laravel-ide.com](https://laravel-idea.com/docs/ide_json) schema and provides
an alias-to-class mapping for PhpStorm with the Laravel Idea plugin. Laravel Idea reads files named
exactly `ide.json`, scans the project recursively, and merges them all — so this file is written to a
package-owned subfolder and is **auto-detected** with no IDE configuration, coexisting with any
`ide.json` your application maintains. Because it is generated inside your application, it reflects
your configured `prefix` and any `merge()`/`replace()` customisations.

Snippets vs. the Extension
--------------------------

The two VS Code outputs serve different audiences:

| | Snippets | Custom Data (via extension) |
|---|---|---|
| **Requires install** | No — picked up automatically from `.vscode/` | Yes — dedicated VS Code extension (forthcoming) |
| **Completion trigger** | Type tag prefix, pick snippet | Type attribute name inside tag |
| **Value completion** | Dropdown of allowed values in snippet | Inline dropdown while typing |
| **Hover docs** | No | Yes |
| **Intended for** | All developers, including those who cannot install extensions | Developers who can install the extension |

Use one or the other — they are not designed to be used together.

Wiring for the Whole Team
-------------------------

Commit the generated files so every developer benefits without having to run the command individually:

```bash
git add .vscode/blade-ui-kit-bootstrap.code-snippets
git add .vscode/blade-ui-kit-bootstrap.html-data.json
git add ide-helper/blade-ui-kit-bootstrap/ide.json
```

To keep the files up to date automatically whenever Composer updates the package, add a `post-update-cmd` script to your `composer.json`:

```json
"scripts": {
    "post-update-cmd": [
        "@php artisan blade-ui-kit-bs:ide --no-interaction"
    ]
}
```

This regenerates all three files after every `composer update`, picking up any new or changed components from the updated package.
